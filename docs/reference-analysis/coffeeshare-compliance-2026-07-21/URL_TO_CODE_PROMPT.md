Use `https://coffeeshare.eu/` as the structural reference for an existing React/Vite frontend, not as a source of copied code or brand assets. The client has supplied local screenshots and a PDF brief; treat those as the primary fidelity targets when the live URL differs.

Capture at matching viewports:

- home, catalog, product detail, blog, contacts;
- desktop 1440×1000 and mobile 390×844;
- header, hero, catalog grid/sidebar, product summary/sidebar, footer and mobile menu.

Then rebuild the current G-Coffee Luxury Showroom pages using CoffeeShare's layout grammar:

- 1200 px container;
- two-tier header;
- 1200×560 image carousel with bottom overlay;
- Lato-like 400/700 typography;
- white canvas, warm brown actions and muted orange active state;
- catalog 840 + 60 + 300 with three 246 px cards;
- product 390 + 360 + 300;
- blog 840 + 300;
- contacts 310 + 716 and 1100×450 map;
- tall three-column dark brown footer.

Preserve G-Coffee's RU/RO routes, MDL, showroom content, filters, search, tabs and indicative calculator. Replace commerce CTAs with consultation/request actions while keeping their spatial role. Use original G-Coffee or newly produced assets; never copy CoffeeShare logo, text, photos, products, code, checkout, PayPal or loyalty flows.

Verify the implementation with side-by-side screenshots at identical viewports. Prioritize visible fidelity of composition, proportions, typography, spacing and card density over adding new decorative ideas.
